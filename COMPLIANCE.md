출시 순서: 🇺🇸 미국 → 🇪🇺 유럽 → 🇰🇷 한국 → 🇯🇵 일본 (App Store Connect에서 국가별 판매 지역 설정으로 제어).

## 접근권한 안내 화면 — 한국만 의무

- 한국: 정보통신망법상 앱 최초 실행 시 접근권한 안내 화면 **의무** → ko 로케일만 노출
- 미국·EU·일본: 별도 화면 요구 없음 (OS의 just-in-time 권한 팝업으로 충족)
- 결론: 권한 안내 화면은 한국어 기기에서만 표시 (구현됨)

## 2. 광고·추적 (ePrivacy / ATT)

- 현재 광고·추적 SDK **없음** → ATT 미요청이 올바른 상태 (불필요 요청은 심사 리젝 사유)
- `requestsAppTracking` 플래그로 구조는 준비됨
- ⚠️ 나중에 광고(AdMob 등) 넣으면: EU는 TCF 동의(UMP SDK), ATT 활성, privacy.html 광고 항목 추가, Apple 프라이버시 라벨 갱신 — 3종 세트가 한 묶음

## 3. 결제 (PaymentGateway 추상화로 이미 유연)

## 4. 콘텐츠·안전 (DSA 등)

- ⚠️ **불법 콘텐츠 신고 기능 필수** (EU DSA + 앱스토어 UGC 정책 + 한국 불법촬영물 대응):
  이모티콘 갤러리·채팅에 "신고" 버튼 → Phase 2 구현 예정
- 신고 접수 창구(이메일) 약관에 명시됨
- 앱스토어 UGC 요건: 신고 + 차단 기능 + 이용약관 동의 — 차단 기능도 Phase 2

## 6. 출시 불가/고위험 국가 (스토어에서 제외 권장)

암호화 메신저 자체가 차단·규제되는 국가: **중국**(외국 메신저 전면 차단), **러시아**(외국 메신저 차단·국영 앱 강제),
**이란·북한**, **UAE**(VoIP 규제), 2025년 **베트남·네팔** 텔레그램 차단 사례 등.
→ App Store Connect 판매 지역에서 제외하면 끝. 이런 국가는 애초에 목록에서 빼고 시작.

또한 프랑스 등 일부 국가는 **암호화 수출 신고**(App Store의 export compliance 질문)가 있는데,
한톡은 표준 HTTPS/TLS만 사용(자체 E2E 암호화 아님)이라 면제 항목(exempt) 해당.

## 7. 수사기관 협조 스탠스 (텔레그램 교훈)

- 서버에 없는 채팅 내용은 제공 불가·의무 없음 (합법)
- **보유한 것(가입정보·접속기록)은 적법한 영장에 협조** — 텔레그램이 2024년까지 전면 비협조로
  n번방·딥페이크 수사 방해 비판과 퇴출론까지 갔다가 정책 전환한 사례가 반면교사
- 약관·정책에 "적법한 절차에 따른 협조" 명시됨 → 이 스탠스가 글로벌 표준
