## 이번 세션에서 한 결정/작업
- **읽음표시 개편**: 1:1은 읽으면 체크 1개, 단톡은 읽은 사람 수. 보낼 땐 표시 없음.
  옵트인(상호) 유지. 데모봇이 읽음신호도 보내 단말 1대로 테스트 가능.
- **채팅 시간 표시**: 같은 사람·같은 분 연속 메시지는 마지막 버블에만.
- **친구추가 UI**: 안드로이드 팝업 → iOS Cupertino 액션시트.
- **알림/진동/사운드 설정** + 방/특정인 음소거(채팅방 종 아이콘). 저장만 됨 —
  실제 푸시 발송은 Firebase(FCM) 연동 시 `NotificationSetting` 값 참조하도록 연결 필요.
- **그림 재생**: 완성본이 기본으로 보이고 ▶로 그리는 과정 재생(예전엔 백지부터 자동재생돼 문제였음).
- **보관 정책 = 사용자 선택**(중요 변경): 기본은 '기기에 계속 보관'. 설정 →
  사라지는 메시지(안 함/24시간/7일). **서버는 어떤 옵션에서도 메시지 미보관**(우체통·ack삭제).
  `RetentionSetting`(기기) + purge 리졸버 방식.
- **사진·동영상 전송**: 아직 미구현. COMPLIANCE.md에 "구현 시 약관·개인정보·권한안내 3종 보강" 리마인더만.

### 그 후 바뀐 것 (원문 대비)
- **iOS 네이티브(Swift) → Flutter 전환.** CocoaPods 배포 → **pub.dev 배포**로 변경.
  Swift 코드는 `ios-native` 브랜치에 보존(미래 네이티브 SDK 뼈대, 지우지 말 것).
- **SwiftData/UserDefaults → sqflite(로컬 DB) + SharedPreferences(설정).**
- **24시간 자동삭제 → 사용자 선택**(기본 보관 / 24시간 / 7일). 서버 미보관은 그대로 유지.
- **그리는 과정 재생 = 서버비용 0으로 해결됨.** 획을 벡터+타임스탬프 JSON으로 보내
  수신 기기가 로컬 재생. (원문의 "DB비용 커질까?" 걱정 → 해결)
- **이모티콘 50원 판매 → 구조만 완성, 플래그 OFF**(`paidEmoticonsEnabled=false`). 지금은 전부 무료.
  업로드/판매는 서버가 appId 결제 확인하는 유료 SDK 옵션.
